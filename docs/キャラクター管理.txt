2. キャラクター管理機能
2.1. 機能の概要と目的
目的: 複数のTRPGシステム（クトゥルフ神話TRPG 6版/7版、エモクロアTRPG、ガイアケアTRPG等）のPCステータスをWeb上で保存・管理する。

主な処理:

システムごとの初期技能や能力値等のマスタに沿った、能力値・技能データの個別保存・上書き。

ココフォリア一発出力機能: 登録データに基づき、ココフォリア等に直接ペーストできるステータス・チャットパレット形式のテキストコマンドを生成しクリップボードに格納する。

所属プレイヤーの紐付け、および通過したシナリオ履歴の自動マッピング。

代理作成: 新規登録時に「プレイヤー」欄で所有者を選べる。characters.player_id は所有者、characters.user_id は作成者（Auth UUID）となり、RLS上の編集権限は作成者側に紐づく（Worker の POST /api/character_full が Service Role で挿入）。

2.2. 画面イメージ（UI構成要素）
キャラクター一覧・検索画面:

登録されているキャラクターのカード型、あるいはリスト型表示。

TRPGシステム別、プレイヤー別、活動状態（生存・ロスト等）別のフィルタリング、および名前や職業のキーワード検索フォーム。

キャラクター詳細画面:

キャラクターの基本情報（名前、読み、職業、年齢、性別、身長、体重、出自、メモ、画像等）。

システムごとのステータス（STR、CONなど）、感情（エモクロア等向けマスタ定義）、および技能値の一覧表示。

「ココフォリア出力」ボタンと、簡易説明を表示する高速ツールチップ。

過去に通過したシナリオ履歴の一覧。

2.3. データ構造（データベース設計）
カラム名,データ型,Nullable,説明
id,text,NO,キャラクターID（主キー）
name,text,NO,キャラクター名
reading,text,YES,キャラクター名のよみがな
system,text,YES,"TRPGシステム名（例: CoC6, Emoklore）"
state,text,YES,状態（生存、ロストなど）
player_id,text,YES,所有プレイヤーのID（外部キー）
user_id,uuid,YES,作成者（Auth UUID）。編集権限の主体
job / age / gender,text / int / text,YES,職業 / 年齢 / 性別
height / weight,integer,YES,身長 / 体重
origin / memo,text,YES,出自 / キャラクターメモ・設定
iachara_url / image_url / race / original_species,text,YES,いあきゃらURL / 画像 / 種族 / オリジナル種
updated_at,timestamptz,NO,更新日時

character_attributes テーブル（能力値）
カラム名,データ型,Nullable,説明
character_id,text,NO,キャラクターID（複合主キー）
key,text,NO,"能力値のキー名（例: STR, HP）（複合主キー）"
value_int,integer,YES,数値型の値
value_emotion,USER-DEFINED,YES,感情データ用（エモクロアTRPG等に対応）
updated_at,timestamptz,NO,更新日時
user_id,uuid,YES,所有者 Auth UUID

character_skills テーブル（習得技能値）
カラム名,データ型,Nullable,説明
character_id,text,NO,キャラクターID（複合主キー）
name,text,NO,"技能名（例: 目星, 調査）（複合主キー）"
value,integer,NO,最終的な合計技能値（％）
skill_key / label,text,YES,システム共通化用の内部キー / 表示ラベル
user_id,uuid,YES,所有者 Auth UUID

関連ビュー: v_character_list / v_character_details（属性・感情・技能・通過シナリオを集約）
