1. プレイヤー（メンバー）管理機能

1.1. 機能の概要と目的
目的: 部内メンバーのプロフィール、TRPGにおけるプレイスタイル（セッションに対する欲求傾向）、スケジュール（空き日程）を一元管理し、卓のセッティングや予定調整を円滑にする。

主な処理:

Discord OAuthを介したソーシャルログインと、部のプレイヤー名簿（players）への紐づけ。

各種欲求パラメータからプレイスタイルをレーダーチャートで可視化。

メンバー間のカレンダー連動による日程の照合（AND計算による空き予定の抽出）。

1.1.1. プレイヤー自己連携（ホーム）
ログイン直後に players.user_id が未設定の場合、ホーム上部に未連携バナーを表示する。

- 名簿に同じ Discord snowflake（players.discord_id）がある場合はサーバー側で自動連携しうる。
- 無い／自動連携できない場合は、user_id 未設定の名簿から自分の名前を選び POST /api/me/link で自己連携する。
- Discord ID はバナーからコピーできる（Auth メタに無い場合は OAuth の provider_token で Discord @me から取得）。
- 開発者向け契約: docs/api-contract.md「プレイヤー自己連携」（GET /api/me・POST /api/me/link）。

1.2. 画面イメージ（UI構成要素）
プロフィール＆プレイスタイル画面:

プレイヤーのアバター（キャラアイコン）、プレイヤー名、自己紹介テキスト。

プレイスタイル傾向を表現するレーダーチャート（Chart.js）。

「推し設定」としてピン留めされたお気に入りキャラクターや、お気に入りシナリオのリスト。

PL通過済 / GM経験済シナリオ一覧（卓履歴から算出）。部活外シナリオはDiscordログインなしでモーダルから登録でき、どのプレイヤーの履歴も追加・削除可能。

GM可能シナリオ一覧（本人のみモーダルで登録）。ここに登録したシナリオで他プレイヤーが「気になる」を押すと Discord DM で通知される。

スケジュール登録・照合画面（カレンダー形式）:

日付ごとに「昼（afternoon）」と「夜（night）」の時間帯スロットを選択し、「〇 (ok)」「△ (maybe)」「× (ng)」を一括入力するフォーム。

複数プレイヤーを指定して日程を照合し、「全員空き（○）」「誰かが不可（×）」「調整中（△と該当者名）」をカラーマッピングで一覧表示するマッチングパネル。

1.3. データ構造（データベース設計）
players テーブル（基本マスタ）
カラム名,データ型,Nullable,説明
player_id,text,NO,プレイヤーの一意の識別ID（主キー）
player_name,text,NO,プレイヤーの表示名
discord_id,text,NO,連携されているDiscordのアカウントID
memo,text,YES,管理者向けメモ
user_id,uuid,YES,Supabase AuthのユーザーUUID
created_at,timestamptz,YES,作成日時

player_profiles テーブル（詳細・傾向データ）
カラム名,データ型,Nullable,説明
player_id,text,NO,プレイヤーID（外部キー）
icon_url,text,YES,アイコンキャラのキャラクターID（列名は歴史的にicon_url）
profile_text,text,YES,自己紹介テキスト
tier_list_first / second / third,text,YES,ティアリスト用テキスト
desire_story,integer,YES,物語欲（0〜5）
desire_avatar,integer,YES,没入欲（0〜5）
desire_harmony,integer,YES,協調欲（0〜5）
desire_chaos,integer,YES,混沌欲（0〜5）
desire_clear,integer,YES,攻略欲（0〜5）
desire_active,integer,YES,主体欲（0〜5）
favorite_character_ids,text[] (ARRAY),YES,お気に入りキャラクターIDの配列（推し設定）
favorite_scenario_ids,text[] (ARRAY),YES,お気に入りシナリオIDの配列（推し設定）
gmable_scenario_ids,text[] (ARRAY),YES,GM可能なシナリオIDの配列（気になる通知の宛先）
external_passed_scenarios,jsonb,NO,部活外通過シナリオの個人履歴（[{id,title,system,note}]、デフォルト []）。scenarios マスタには載せない
created_at / updated_at,timestamptz,YES,作成・更新日時

player_availability テーブル（予定データ）
カラム名,データ型,Nullable,説明
player_id,text,NO,プレイヤーID（複合主キー）
target_date,date,NO,対象日（複合主キー）
time_slot,text,NO,時間帯（afternoon / night、複合主キー）
status,text,NO,空き状況（ok / maybe / ng）
raw_text,text,YES,システムやユーザーによるメモ（例: 占有時自動テキスト）
user_id,uuid,YES,所有者 Auth UUID
created_at / updated_at,timestamptz,NO,作成・更新日時
